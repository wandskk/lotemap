# Resumo de Decisões

## 1. UI
**Decisão:** shadcn/ui + Tailwind  
**Motivo:** velocidade, consistência, bom encaixe com Next.js.

## 2. Mapa
**Decisão:** SVG interativo  
**Motivo:** melhor para plantas de loteamento, menor complexidade que GIS.

## 3. Hospedagem
**Decisão:** Vercel  
**Motivo:** excelente para Next.js e páginas públicas.

## 4. Banco
**Decisão:** PostgreSQL com Prisma  
**Motivo:** maturidade, consistência relacional, produtividade.

## 5. Contato comercial
**Decisão:** WhatsApp por loteamento  
**Motivo:** simples, direto e suficiente para conversão no MVP.

## 6. Multi-tenant
**Decisão:** modelar desde o início  
**Motivo:** evita retrabalho estrutural.
